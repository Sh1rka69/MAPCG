MAPCG Editor Guide

Sections:

About the Editor

MAPCG is a browser-based 3D map editor for the game GoreBox. It lets you build levels out of cubes and primitive shapes, set up materials, lighting and weather, then export the finished map straight into the GoreBox format (ZIP / MapData.json).

Developer
Sh1rka

Editor Basics

MAPCG is a cube-grid based map editor for GoreBox. Maps are built from cubes, primitive shapes (cylinders, triangles, rhombi, spheres) and imported 3D models. Every object is edited through the panel on the right: material, texture, physics, color.

Cube — the core building block. Can be stretched, painted, and re-textured per face.
Spawn Point — where the player appears on the map once it starts.

Transform Tools

Translate
Moves the selected object along the X/Y/Z axes. Key G.
Rotate
Rotates the object around its axes. Key R.
Scale
Resizes the object along its axes. Key T.
Snap to Grid
Constrains movement and size to grid steps so cubes align cleanly.
Undo / Redo
Undo and redo recent actions: Ctrl+Z / Ctrl+Y.

Shapes & Model Import

Import 3D (OBJ / GLB / GLTF) — your own model gets converted into cubes using a "1 polygon = 1 cube" approach.
Whole Shape — a mode where clicking any cube of a shape selects the entire shape at once (handy for moving/painting composite objects in one go).
Texture Selection — when enabled, clicking a cube selects all cubes with the same texture. If Whole Shape is also enabled and the clicked cube belongs to a group, only cubes with that texture inside the same group are selected.

Materials & Textures

Cube Material (Tag)
The texture applied to a cube's face: concrete, wood, metal, carpet, grass, glass (transparent, reflective), plasma (emissive, with bloom), and more.
Texture Tiling
How a texture repeats on a face: Stretch (fills the whole face), XY / YX (1×1 m tiling in different directions).
Physics Material
The surface's in-game physical behavior (friction, bounce, etc.) — affects gameplay, not visuals.
Color (Tint)
A color tint layered on top of the cube's texture.

Map Settings

Sky & Weather
Skybox (day/dawn/sunset/night/midnight/custom image), sun angle (horizontal/vertical), and weather: clear, cloudy, rain, storm, snow, blizzard, fog.
Interface
UI scale, camera sensitivity and speed, hitbox opacity, and mobile joystick settings (mode, opacity, size).
Graphics
Dynamic shadows, Bloom (glow), Corner Shadows (SSAO), Ray Tracing, water waves, texture/pixel smoothing, and an FPS counter. Quality is automatically reduced on mobile devices.
Fog
Fog color (in clear weather) plus the start and end distances for visibility falloff.

Collection of Subjects

A ready-made set of objects (for example, from the Vabish library) that can be quickly placed onto the map.

+ Add Collection — load your own object set from a ZIP archive.
Select → Panel — the selected collection object opens in the editing panel.
✦ Spawn Object — places the selected collection object directly onto the map.

Importing & Exporting Maps

Import ZIP Map
Loads a GoreBox map: MapData.json + the CustomTextures folder.
Import .gbmap
Legacy GoreBox map format (both V15 and V16 are supported, including gzip/ZIP/UTF-16 encodings).
Save ZIP / Export with Lighting
Packs the finished map into a ZIP archive (MapData.json v4 + textures). The "with Lighting" option additionally bakes lighting per cube face.
Export to GLB
Exports the map as a 3D model for viewing or further editing in Blender and other tools.

Useful Tips

  • •Save regularly (Ctrl+S) — especially before baking lighting or a heavy import.
  • •On mobile devices, graphics (shadows, reflections, ray tracing) are automatically simplified for stable FPS.
  • •Use Whole Shape so you don't have to select a composite shape cube by cube. Use Texture Selection when you need to quickly select/paint only cubes with one material; combined with Whole Shape it is limited to the clicked object's group.
  • •Before exporting to GoreBox, make sure the spawn point is set and not stuck inside other cubes.
